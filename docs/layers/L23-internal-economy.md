# Layer 23 — Internal Economy (AIX)

**Core innovation:** Closed-loop token economy for resource allocation.

## Scope

A non-convertible internal currency — **AIX (Agent Internal eXchange)** — that agents earn by producing validated outcomes (profitable signals, published principles, successful architectural proposals) and spend on resources (CPU, GPU, memory, data subscriptions, compute quotas, LLM tokens, priority bus bandwidth).

## Earning Rules

- Profitable signal: +AIX proportional to risk-adjusted P&L.
- Published principle (Gold Node): +AIX scaled by citation count over time.
- Approved architecture proposal: one-time +AIX bonus from Narcissus.
- Approved advancement: +AIX drawn from a per-milestone reserve pool.
- Heartbeat uptime: micro-rewards for availability.

## Spending Rules

- CPU hours, GPU hours, RAM reservation — priced by resource-balancer (Hephaestus).
- Data vendor query — priced by data-vendor-negotiator (Agora).
- LLM token budget — priced by provider, marked up for overhead.
- Priority bus slot — variable, auction-based during contention.
- Dream Room session — fixed per minute of accelerated replay.

## Acceptance Criteria

- AIX ledger is itself an append-only hash-chained store; every transfer is signed.
- Insolvent agent → automatic sleep until balance restored; repeated insolvency → reaper.
- Supply schedule documented; no agent can mint.
- Dashboard view shows per-city GDP, per-agent wealth distribution, and liquidity.
